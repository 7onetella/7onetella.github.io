    Ansible

    Context

    I tried learning about Puppet and Chef a while back. I was trying to use Puppet or Chef to update running Vagrant machines. That was more for setting up local development environment.

    Docker and ECS came around next. That improved application deployment. Docker solved two problems. Local development environment setup and application deployment in the cloud.

    Since then there was no need for me to go back to learning about Puppet and Chef until now.

    Ansible is another tool simiar to Puppet or Chef. What made it attractive to use Ansible is #1 its easy of use, #2 its integration with Aminator.

    I use Ansible together with Aminator to bake lots of AMIs, including

    • Consul
    • Fabio LB
    • Aminator
    • Jenkins
    • Java 8
    • Vim-Go

    I even have Ansible playbook for setting up remote server vim-go environment.

    All my playbooks are on github repo
