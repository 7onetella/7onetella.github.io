    The story behind docker image "alpinegovim"

    Motivation

    Many of you might think that wanting to develop remotely in docker container is bazaar. Here is why alpinegovim was created. alpinegovim is all-in-one development environment which can run in a docker container in the cloud with access to terminal as a web application.

    Beginning

    One day I was working for a company in NY as a consultant. Consulting agreement was signed in advance. Project manager had put in a request for the laptop I said I would need. I would say there was at least 2 months lead time for helpdesk to procure the equipment I needed.

    Problem

    It’s the first day of work, I am told it will take about 1 month and a half to for me to get the laptop. My consulting contract was for 6 months. My predicament was that I didn’t have all the time in the world for me to wait for the laptop and then rush to finish the project. I was given a very old laptop with under powered CPU and memory while I waited for the laptop.

    Solution

    I came up with an idea to use remote server for development. I used remote desktop like software called NoMachine to remotely access a powerful server. I installed all my development tools and started my development there. The project was successfully completed on time. At this point, you might be asking which programming language would require so much CPU power and memory to develop. The client company’s IT shop’s choice of language was Java. If you know Java, you know Java is heavy in memory usage.

    Now with Golang

    Ever since then I have discovered Go. I will do Python if required. I will do Nodejs as well. All of these three languages are light. You can simply use any text editors to develop. I would prefer to do Go if I am given the freedome to choose the language to develop in.

    Vim-go

    If anyone has watched “Vim-go” YouTube video, you know anyone can be very productive writing code in vim once you have mastered vim. Vim-go just makes writing Go code so much fun and easy.

    Reality

    I want to be honiest. I have NOT master vim yet. I would like to have my laguages and tools all installed and ready to go in seconds just in case if I am ever again thrown into a situation where I have no choice but to develop on a remote server. I am hoping that this might happen sometime in the future. I would be forced to do all my development on vim. That would be a fantastic way to master vim-go!

    As of today

    I have yet to use “alpinegovim” on a remote server. I am very happy with vscode. However, I have used gotty to remotely access my docker instance’s terminal in “web browser” and troubleshoot. Life is good.
